# UI 및 컴포넌트 지침

## 범위

- 이 지침은 `app/` 아래의 React 컴포넌트, 페이지, 스타일 변경에 적용한다.

## 컴포넌트 구조

- 계산기, PracticeInputs, TodoList 예제는 서로 독립적인 컴포넌트로 유지한다.
- 학습용 예제는 기능이 한눈에 보이도록 단순하게 작성한다.
- 상태 변경 로직은 컴포넌트 안에서 읽기 쉬운 함수로 분리한다.

## 테스트 식별자

- 테스트 학습 편의를 위해 주요 입력, 버튼, 결과, 목록 항목에는 `data-testid`를 사용할 수 있다.
- `data-testid` 값은 컴포넌트 이름이나 기능 이름을 앞에 붙여 충돌을 피한다.
  예: `practice-result`, `todo-add-button`, `todo-item`.

## 디자인

- 입력, 버튼, 결과 영역은 사용자가 역할을 쉽게 파악할 수 있게 배치한다.
- 과한 장식보다 테스트와 학습 흐름이 잘 보이는 명확한 UI를 우선한다.
- 텍스트가 작은 화면에서 겹치거나 잘리지 않도록 여백과 줄바꿈을 고려한다.

## 반응형 기준

- 모바일 지원 범위는 370px부터 500px까지로 정의한다.
- Tailwind `sm` breakpoint는 501px(`31.3125rem`)부터 적용하며, `app/globals.css`의 `@theme inline`에서 중앙 관리한다.
- 모바일 스타일은 base 클래스에 작성하고 501px 이상에서 필요한 확장 스타일을 `sm:` variant로 작성한다.
